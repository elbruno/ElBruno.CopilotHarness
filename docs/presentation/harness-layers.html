Click or press → to reveal each layer of the harness.
Copilot — your everyday AI assistant in VS Code, the Copilot App, or CLI. It sends requests that look like standard OpenAI API calls.
BYOK Configuration — in VS Code's Manage Language Models, point the custom endpoint at your harness URL. Copilot routes through your infrastructure, not GitHub's.
Router (Aspire) — aspire run spins up Router.Api (the OpenAI-compatible proxy), Admin.Web (the operator dashboard), and the Aspire telemetry dashboard as a single unit.
Model Selector — the Rules Engine evaluates conditions (keyword, regex, prompt size, intent, streaming) against an ordered rule list. An AI processor classifies intent; every decision is logged with a human-readable explanation.
Models — small fast prompts stay local (Ollama llama3.1:8b, private, free). Large or agentic requests go to Azure OpenAI / gpt-5-mini. You control which model handles what.
Why it matters

Your Copilot, Your Rules

GitHub Copilot is powerful — but every prompt goes to a single cloud model you don't control.
The harness puts you back in charge.

🔒 Privacy — local LLM for sensitive code 💰 Cost — free local inference where it fits 🔎 Explainability — see every routing decision live ⚖️ Control — condition-based rules you write 📊 Evaluation — AI Judge benchmarks model quality
Getting Started

Three commands, then you're routing

1 — Set secrets (once)
aspire secret set FoundryEndpoint "https://…"
aspire secret set FoundryApiKey "<key>"
2 — Launch
aspire run
3 — Connect Copilot (BYOK)
VS Code → Manage Models → Custom Endpoint
URL: http://localhost:5117/v1/chat/completions
